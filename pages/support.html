快速入门

Quick Start

1

启动应用

打开 DOMPrompter，输入目标页面 URL 或从开发服务器启动。应用自动通过 Chrome DevTools Protocol 连接页面。

Launch App

Open DOMPrompter, enter the target page URL or launch from your dev server. Auto-connects via Chrome DevTools Protocol.

2

选择元素

在目标页面点击任意元素。蓝色边框高亮选中容器，浮动动作按钮出现在蓝框周围。

Select Elements

Click any element on the target page. Blue outline highlights the container, floating action buttons appear around the frame.

3

调整属性

蓝色 W/H 按钮调整宽高，绿色按钮调整 padding，橙色按钮调整 margin。也可在右侧属性面板精确输入数值。

Adjust Properties

Blue W/H buttons adjust size, green for padding, orange for margin. Or enter precise values in the properties panel.

4

添加标签

为元素添加文字标签描述修改意图。标签徽章附着在对应元素上，支持多个元素同时标注。

Add Tags

Add text tags describing your change intent. Tag badges attach to the element. Multiple elements can be annotated simultaneously.

5

导出提示词

点击导出，所有选择器 + 样式差异 + 标签注释合并为结构化 AI 提示词。复制粘贴到 Cursor / Claude Code / Codex 即可。

Export Prompt

Click export — all selectors + style diffs + tag annotations merged into a structured AI prompt. Copy-paste into Cursor / Claude Code / Codex.

常见问题

FAQ

DOMPrompter 支持哪些浏览器？ Which browsers does DOMPrompter support?

通过 Chrome DevTools Protocol (CDP) 工作，支持 Chrome 和所有 Chromium 内核浏览器（Edge、Brave 等）。

Works via Chrome DevTools Protocol (CDP), supporting Chrome and all Chromium-based browsers (Edge, Brave, etc.).

DOMPrompter 会修改我的源代码吗？ Does DOMPrompter modify my source code?

不会。DOMPrompter 只修改浏览器中的运行时 DOM，不直接修改源代码。所有调整是临时的页面内修改，刷新后恢复。它的作用是生成精确的 AI 提示词，由你手动交给 AI 工具修改源代码。

No. DOMPrompter only modifies the runtime DOM in the browser — never touches source files. All adjustments are temporary, resetting on refresh. Its role is generating precise AI prompts for you to hand to AI tools.

可以同时标注多个元素吗？ Can I annotate multiple elements at once?

可以。依次选择不同元素并添加标签和样式调整。导出时所有已标注元素的数据合并为一个完整的 AI 提示词。

Yes. Select different elements one by one and add tags/adjustments. On export, all annotated elements merge into one complete AI prompt.

如何撤销样式调整？ How do I undo style adjustments?

Cmd+Z 撤销，Cmd+Shift+Z 重做。所有通过浮动按钮或属性面板的调整都记录在操作历史中。

Cmd+Z to undo, Cmd+Shift+Z to redo. All adjustments via floating buttons or properties panel are recorded in history.

应用窗口消失了怎么办？ What if the app window disappears?

在 macOS Dock 中点击 DOMPrompter 图标恢复窗口。如仍无法显示，尝试重启应用。

Click the DOMPrompter icon in macOS Dock to restore. If still not visible, restart the app.
